<div xmlns:xlink="http://www.w3.org/TR/xlink" xmlns:m="http://www.loc.gov/METS/" xmlns:cdl="http://ark.cdlib.org/schemas/appqualifieddc/" xmlns:dc="http://purl.org/dc/elements/1.1/" xmlns:dcterms="http://purl.org/dc/terms/" class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#METS_Profile">METS_Profile:</a>
@xsi:schemaLocation="http://www.loc.gov/METS_Profile/ http://www.loc.gov/standards/mets/profile_docs/mets.profile.v1-2.xsd     http://www.loc.gov/METS/  http://www.loc.gov/standards/mets/mets.xsd      http://www.loc.gov/mods/ http://www.loc.gov/standards/mods/v3/mods-3-0.xsd"
<br>
    <div><a href="http://www.loc.gov/mets/profiles/00000001.xml">URL: http://www.loc.gov/mets/profiles/00000001.xml</a></div>
    <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#title">title:</a><br>Oxford Digital Library METS Profile</div>
    <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#abstract">abstract:</a><br>This profile documents the use of METS for standard image or image+text digital
        objects within the Oxford Digital Library (ODL). </div>
    <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#date">date:</a><br>2003-12-12T15:15:00</div>
    <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#contact">contact:</a><br>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#name">name:</a><br>Richard Gartner</div>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#address">address:</a><br>SERS Building, Osney Mead, Oxford OX2 0EZ, UK</div>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#phone">phone:</a><br>+44 1865 280029</div>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#email">email:</a><br>richard.gartner@sers.ox.ac.uk</div>
    </div>
    <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#related_profile">related_profile:</a><br>There are no related profiles.</div>
    <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#extension_schema">extension_schema:</a><br>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#name">name:</a><br>MODS (Metadata Object Description Schema) schema, version 3</div>
        <div><a href="http://www.loc.gov/standards/mods/v3/mods-3-0.xsd">: http://www.loc.gov/standards/mods/v3/mods-3-0.xsd</a></div>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#context">context:</a><br>Used for encoding descriptive metadata</div>
    </div>
    <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#extension_schema">extension_schema:</a><br>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#name">name:</a><br>ODL Administrative Metadata Schema</div>
        <div><a href="http://www2.odl.ox.ac.uk/data/aaaaaa/schema/odl-adminmetadata.dtd">: http://www2.odl.ox.ac.uk/data/aaaaaa/schema/odl-adminmetadata.dtd</a></div>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#context">context:</a><br>Used to encode administrative metadata for image files. This DTD is used only for
            external metadata records, and should only be used for records linked from a techMD element.</div>
    </div>
    <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#extension_schema">extension_schema:</a><br>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#name">name:</a><br>ODL extensions to MODS</div>
        <div><a href="http://www2.odl.ox.ac.uk/data/aaaaaa/schema/odl.xsd">: http://www2.odl.ox.ac.uk/data/aaaaaa/schema/odl.xsd</a></div>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#note">note:</a><br>Extensions to the MODS schema specific to the ODL: these cover any facets of
            descriptive metadata not adequately handled by MODS itself</div>
    </div>
    <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#description_rules">description_rules:</a><br>
        <p>Descriptive metadata should follow the ODL Guidelines for descriptive metadata
            (http://www2.odl.ox.ac.uk/guidelines/), including the ODL's name authority rules
            enumerated within these (essentially conformance to the Anglo-American Name Authority
            file with supplementary lists specific to the ODL for names absent from this). The
            metadata is input into the ODL's webform-based metadata system using qualified Dublin
            Core fields (described in the above guidelines), but converted to the MODS schema when
            the METS file is generated.</p>
        <p>Any names or titles not taken from an authority list should be formatted according to
            AACR2 conventions.</p>
    </div>
    <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#controlled_vocabularies">controlled_vocabularies:</a><br>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#vocabulary">vocabulary:</a><br>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#name">name:</a><br>Library of Congress Subject Headings</div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#maintenance_agency">maintenance_agency:</a><br>Library of Congress Network Development &amp; MARC Standards Office</div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#context">context:</a><br>
                <p>At least one LCSH heading is compulsory for each record - other taxonomies
                    specific to each project may be used in addition to this</p>
            </div>
        </div>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#vocabulary">vocabulary:</a><br>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#name">name:</a><br>MARC21 relator codes</div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#maintenance_agency">maintenance_agency:</a><br>Library of Congress Network Development &amp; MARC Standards Office</div>
            <div><a href="http://www.loc.gov/marc/relators/relaterm.html">: http://www.loc.gov/marc/relators/relaterm.html</a></div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#context">context:</a><br>
                <p>Used for encoding the relationship of any person or entity named in descriptive
                    metadata to the item itself.</p>
            </div>
        </div>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#vocabulary">vocabulary:</a><br>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#name">name:</a><br>ODL List of Types</div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#maintenance_agency">maintenance_agency:</a><br>Oxford Digital Library</div>
            <div><a href="http://www2.odl.ox.ac.uk/guidelines/odl-types.html">: http://www2.odl.ox.ac.uk/guidelines/odl-types.html</a></div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#context">context:</a><br>
                <p>Used to encode the genre of the item</p>
            </div>
        </div>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#vocabulary">vocabulary:</a><br>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#name">name:</a><br>ISO 639-2</div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#maintenance_agency">maintenance_agency:</a><br>International Standard Organization</div>
            <div><a href="http://www.iso.ch/">: http://www.iso.ch/</a></div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#context">context:</a><br>
                <p>Encodes the languages used in the content of the item</p>
            </div>
        </div>
    </div>
    <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#structural_requirements">structural_requirements:</a><br>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#metsHdr">metsHdr:</a><br>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>The OBJID attribute in the mets element must be present: this contains the system
                    id for the item (eg. munahi010-aaa).</p>
            </div>
        </div>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#dmdSec">dmdSec:</a><br>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>One dmdSec, identified by an ID attribute in the form "objectid-dmd-0001" will
                    contain a MODS record detailing the descriptive metadata for the item as a
                    whole: this is embedded in a mdWrap element.</p>
            </div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>Other dmdSecs may contain descriptive metadata in other schemas for all or part
                    of the item: these are referenced by DMDID attributes within the structMap.</p>
            </div>
        </div>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#amdSec">amdSec:</a><br>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>Files are grouped according to their referent into amdSec elements, one for each
                    object referenced: thus, all adminstrative metadata for manifestations of the
                    same scanned object (eg page of a volume) are grouped into the same amdSec.</p>
            </div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>Each amdSec is given an identifier in the form "objectid-amd-0001" etc, numbered sequentially.</p>
            </div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>Each amdSec contains one techMD element for each file associated with the object
                    referenced by it: these techMD elements are given identifiers in the form
                    "objectid-tmd-0001-0" numbered sequentially - the final digit refers to the
                    version of the referent instantiated by the file, according to the same schema
                    as that followed by the file's GROUPID attribute in the fileSec.</p>
            </div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>Each techMD element contains an mdRef element referencing the location of the XML
                    file containing the administrative metadata itself. The MDTYPE attribute is set
                    to "OTHER" and the OTHERMDTYPE attribyte to "ODL Admin Metadata Scheme".</p>
            </div>
        </div>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#fileSec">fileSec:</a><br>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>Files are grouped according to their referent into fileGrp elements: thus, all
                    manifestations of the same scanned object (eg page of a volume) are grouped into
                    the same fileGrp.</p>
            </div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>Each fileGrp is given an identifier in the form "objectid-fgrp-001" etc, numbered sequentially.</p>
            </div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>The GROUPID attribute for each file element refers to which instantiation of the
                    referent is represented by the given file - eg thumbnail, delivery version. The
                    following scheme is used for these codes: 0 = archival master, 1 = archival
                    production, 2 = delivery, large, 3 = delivery, medium, 4 = delivery, small, 5 =
                    thumbnail, 6 = small thumbnail, 7 = other. Non-image files may be identified by
                    an suitable code, eg TEI. </p>
            </div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>An fLocat element within each file element references the file itself by the
                    xlink:href attribute.</p>
            </div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>An ADMID attribute on each file attribute points to the technical metadata for
                    that file held within the admSec.</p>
            </div>
        </div>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#structMap">structMap:</a><br>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>Only one structMap is present in the METS document: this contains both the
                    logical and physical structures of the item within a single hierarchy. This
                    hierarchy is bounded by a div element, the DMDID attribute of which is set to
                    the ID "objectid-dmd-0001" - ie. the ID of the MODS record describing the item
                    as a whole. </p>
            </div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>The div elements within the structural map each have an ID attribute in the form
                    "objectid-div.x.x.x." where "x.x.x" represents a hierarchical numbering system
                    which mirrors the hierarchical arrangement of the divs themselves (eg munahi010-aaa-div.1.2.1)</p>
            </div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>No type attributes are used for div elements</p>
            </div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>Each lowest level div element contains a fptr element, whose FILEID attribute
                    points to the fileGrp of the referenced item in the fileSec.</p>
            </div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>A LABEL attribute must be provided for each div element.</p>
            </div>
        </div>
    </div>
    <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#technical_requirements">technical_requirements:</a><br>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#content_files">content_files:</a><br>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>Archival quality images, identified by GROUPID=0 in the file element, are scanned
                    in TIFF format, in bit-depths of either 1,8 or 24 bits per pixel depending on
                    the requirements of the source material. 1 bit images are compressed using
                    G4(2D) compression, otherwise the images are uncompressed.</p>
            </div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>Medium-sized delivery images, identified by GROUPID=3, are stored in JPEG format
                    at a maximum resolution of 1000x1000 pixels. </p>
            </div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>Thumbnail images, identified by GROUPID=6, are stored in JPEG format at a maximum
                    resolution of 96x96 pixels.</p>
            </div>
            <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#requirement">requirement:</a><br>
                <p>All text files referenced by the METS object must conform to the TEI P4 DTD
                    (http://www.tei-c.org/P4X/) and to one of levels 1-4 of the TEI Text Encoding in
                    Libraries Guidelines for Best Encoding Practices
                    (http://www.indiana.edu/~letrs/tei/). </p>
            </div>
        </div>
    </div>
    <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#tool">tool:</a><br>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#agency">agency:</a><br>Oxford Digital Library</div>
        <div class="gx">
<a href="../profile_docs/mets.profile.v1-2.html#description">description:</a><br>
            <p>METS files for the Oxford Digital Library are created by an in-house web-form input
                mechanism, based on mySql and PHP. It is not available for general access at present.</p>
        </div>
    </div>
