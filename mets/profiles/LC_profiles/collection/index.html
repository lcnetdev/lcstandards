Library of Congress METS Profile for Collections [Draft]

XML Version of Profile Document



Abstract

The Library of Congress METS Profile for collections specifies how a METS document representing a collection of digital library objects (which are in turn represented as METS instances) should be encoded for inclusion in a Library of Congress digital library project. This profile is useful for aggregating any arbitrary group of documents into a digital library collection. The profile uses an "explicit" method of aggregation (rather than "implicit" or "rule-based" method) in that it provides an explicit list of the items to be included in the collection.

Controlled Vocabularies

Structural Requirements

Descriptive Metadata Section
dmdSec Requirement 1

A document must contain one and only one Descriptive Metadata Section (dmdSec) with a wrapped (mdWrap) MODS bibliographic record. The mods element must have an ID attribute. This MODS record provided a collection-level description to a group of documents.

Structure Map
structMap Requirement 1

The physical structure of the collection will be represented in the Structure Map (structMap) section of the METS instance. The document will have one and only one structMap. The structMap element will have one and only one top level div TYPE="coll:collectionObject" child element. The top-level div element of the structMap will have a DMDID attribute that points to the mods element.
