LC METS Profile

Library of Congress METS Profile for Bibliographic Records

XML Version of Profile Document


Abstract

The Library of Congress METS Profile for Bibliographic Records specifies how METS documents representing bibliographic records within Library of Congress digital library projects should be encoded. This profile is intended to be used in cases where there is a need to include bibliographic records for items that have not yet been digitized in a digital library project. The profile may also be used in cases where there is a need to associate a partially digitized item with the bibliographic data. The profile provides for three cases of partially digitized items, 1) a bibliographic record with an associated "illustrative" digital image (e.g. an image of a book cover), 2) a bibliographic record with a digital version (either images or text) of a table of contents, or 3) a bibliographic record with any other digitized segment of the item. In addition, the profile may be used in cases where there is a bibliographic record with a digitized version of corresponding catalog cards. It is interesting to note that this METS Profile makes it possible to combine records for non-digitized documents or partially digitized documents into the same XML datastore with digitized documents, using METS as a common data structure.

profile:dmdSec Requirement 2

The mods record may contain one or more relatedItem type="constituent" child elements which describes a sub-part or excerpt (e.g. the table of contents from a book) of the item. Note that the structMap contains a div TYPE="bib:toc" element that points to the corresponding relatedItem type="constituent" element.

File Section
profile:fileSec Requirement 1

In cases where there are no associated digital files (for an illustrative image, a table of contents, or for digitaized catalog cards, etc.) there will be no fileSec.

In cases where there are associated digital files, those files should be represented in the fileSec as in the following example. Note that master files and service (or derivative files) are placed in separate fileGrp sections which have an attribute USE="MASTER" or USE="SERVICE". Note also that the fptr elements in the structMap have FILEID attributes that point to the corresponding file elements in the fileSec.

Example 3.

    <mets:mets xmlns:bib="http://www.loc.gov/mets/profiles/bibRecord" xmlns="http://www.loc.gov/METS/" xmlns:rights="http://www.loc.gov/rights/" xmlns:lc="http://www.loc.gov/mets/profiles" PROFILE="lc:bibRecord">

      <mets:dmdSec ID="dmd01_ex03">
        <mets:mdWrap MDTYPE="MODS">
          <mets:xmlData>
            <mods:mods ID="mods1_ex03">
              <mods:titleInfo>
                <mods:title>I'm going back to Georgia</mods:title>
              </mods:titleInfo>
            </mods:mods>
          </mets:xmlData>
        </mets:mdWrap>
      </mets:dmdSec>
      <mets:fileSec>
        <mets:fileGrp USE="MASTER">
          <mets:file MIMETYPE="image/tiff" GROUPID="G1" ID="f_ex03m">
            <mets:FLocat LOCTYPE="URL" xlink:href="http://lcweb4.loc.gov/natlib/ihas/warehouse/afc9999005/AFS_1500_A-1839_B3/0850.tif" />
          </mets:file>
        </mets:fileGrp>
        <mets:fileGrp USE="SERVICE">
          <mets:file MIMETYPE="image/jpeg" GROUPID="G1" ID="f_ex03s">
            <mets:FLocat LOCTYPE="URL" xlink:href="http://lcweb4.loc.gov/natlib/ihas/service/afc9999005/AFS_1500_A-1839_B3/0850v.jpg" />
          </mets:file>
        </mets:fileGrp>
      </mets:fileSec>
      <mets:structMap>
        <mets:div DMDID="mods1_ex03" TYPE="bib:bibRecord">
          <mets:div TYPE="bib:card">
            <mets:div TYPE="lc:image">
              <mets:fptr FILEID="f_ex03m" />
              <mets:fptr FILEID="f_ex03s" />
            </mets:div>
          </mets:div>
        </mets:div>
      </mets:structMap>
    </mets:mets>

profile:structMap Requirement 2

The profile provides for three cases of partially digitized items, 1) a bibliographic record with an associated "illustrative" digital image (e.g. an image of a book cover), 2) a bibliographic record with a digital version (either images or text) of a table of contents, and 3) a bibliographic record with any other digitized segment of the item. In the first case, an illustrative image will be represented in the structMap as a div TYPE="bib:illustration" element. In the second case, a digitized table of contents will be represented as a div TYPE="bib:toc" elements. In the third case, some other digitized segment will be represented as a div TYPE="bib:other". It is recommended, in this case, that the LABEL attribute be used to supply information about the nature of the digitized segment. The profile also provides for cases where there is a bibliographic record with a digitized version of corresponding catalog cards. In all cases where there are digital files the structMap will contain a div TYPE="lc:image" or div TYPE="lc:text" element as needed. Each div TYPE="lc:image" or div TYPE="lc:text" element will have a child fptr element with a FILEID attribute pointing to the corresponding file element in the fileSec. This is illustrated above in Example 3.

Example 5.

    <mets:mets>
      <mets:dmdSec ID="dmd01_ex05">
        <mets:mdWrap MDTYPE="MODS">
          <mets:xmlData>
            <mods:mods ID="mods1_ex05">
              <mods:titleInfo>
                <mods:title>Title</mods:title>
              </mods:titleInfo>
            </mods:mods>
          </mets:xmlData>
        </mets:mdWrap>
      </mets:dmdSec>
      <mets:structMap>
        <mets:div TYPE="bib:bibRecord" DMDID="mods1_ex05">
          <mets:div TYPE="bib:illustration">
            <mets:div TYPE="lc:image">
              <mets:fptr />
            </mets:div>
          </mets:div>
          <mets:div TYPE="bib:toc">
            <mets:div TYPE="lc:image">
              <mets:fptr />
            </mets:div>
            <mets:div TYPE="lc:image">
              <mets:fptr />
            </mets:div>
          </mets:div>
          <mets:div TYPE="bib:toc">
            <mets:div TYPE="lc:text">

Technical Requirements

Content Files

Still Image Files

The master image files referenced by conforming documents must be in TIFF Revision 6.0 format. See Digital Formats for Library of Congress Collections: TIFF, Revision 6.0 at http://www.digitalpreservation.gov/formats/fdd/fdd000022.shtml.

Text Files

Master text files referenced by conforming documents must be in XML format. See Digital Formats for Library of Congress Collections: XML (Extensible Markup Language) at http://www.digitalpreservation.gov/formats/fdd/fdd000075.shtml.
